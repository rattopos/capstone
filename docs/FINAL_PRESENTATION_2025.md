## 1. 프로젝트 개요

### 프로젝트 목적

> **국가데이터처 지역경제동향 보도자료의 자동 생성 시스템 구축**

- 분기별 발행되는 지역경제동향 보도자료를 자동으로 생성
- 기초자료 수집표 업로드만으로 약 **50개 이상의 보도자료**를 3분 내에 생성
- 실무팀의 단순 반복 작업을 자동화하여 분석 업무에 집중할 수 있도록 지원

### 프로젝트 배경

| 구분 | 내용 |
|------|------|
| **발행 기관** | 국가데이터처 (Ministry of Data and Statistics) |
| **발행 주기** | 분기별 (연 4회) |
| **실무 팀** | 담당 사무관 1명 + 주무관 1명 (총 2명) |
| **기존 작업 기간** | 약 7일 (초안 5일 + 해석 2일) |
| **대상 보도자료** | 약 50개 이상 |

## 2. 문제 인식

### 2.1 기존 업무 프로세스의 문제점

```
┌─────────────────────────────────────────────────────────────┐
│                    기존 업무 프로세스                        │
├─────────────────────────────────────────────────────────────┤
│  기초자료 수집 → 엑셀에서 데이터 추출 → 수동 타이핑 →       │
│  한글(HWP) 문서 작성 → 검토 및 수정 → 최종 발행            │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 주요 문제점

| 문제 | 설명 | 영향 |
|------|------|------|
| ⏰ **시간 소모** | 보도자료 초안 작성에만 5일 소요 | 해석 업무 시간 부족 |
| 🔄 **반복 작업** | 엑셀 → 한글 복사-붙여넣기, 수동 타이핑 | 업무 효율 저하 |
| ⚠️ **인적 오류** | 수동 입력 과정에서 실수 발생 | 품질 저하 |
| 😓 **업무 피로도** | 단순 반복 작업으로 인한 피로 누적 | 실무자 만족도 저하 |
| 📊 **해석 시간 부족** | 초안 작성에 시간을 많이 사용 | 분석의 깊이 한계 |

### 2.3 핵심 Pain Point

> **"초안 작성에 5일을 쓰다 보니, 정작 중요한 해석 작업에는 2일밖에 투자할 수 없다."**

## 3. 해결 방안

### 3.1 시스템 도입을 통한 자동화

```
┌─────────────────────────────────────────────────────────────┐
│                    개선된 업무 프로세스                      │
├─────────────────────────────────────────────────────────────┤
│  기초자료 수집표 업로드 → [자동] 데이터 추출 →              │
│  [자동] 보도자료 생성 → 미리보기 및 검토 → 최종 발행       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 해결 전략

1. **자동화**: 엑셀 데이터 추출 및 보도자료 생성 자동화
2. **템플릿화**: Jinja2 기반 HTML 템플릿으로 일관된 형식 유지
3. **결측치 처리**: 비공개 자료는 실무자가 직접 입력하도록 설계
4. **검토 시스템**: 각 보도자료별 검토 완료 상태 추적
5. **일괄 생성**: 모든 보도자료를 한 번에 생성

## 5. 주요 기능

### 5.1 핵심 기능 목록

| 기능 | 설명 |
|------|------|
| 📁 **데이터 업로드** | 드래그 앤 드롭 또는 클릭으로 기초자료 수집표 업로드 |
| 📅 **연도/분기 자동 감지** | 업로드된 파일에서 최신 연도/분기 자동 감지 |
| 👁️ **실시간 미리보기** | 각 보도자료 HTML을 실시간으로 확인 |
| ◀️▶️ **페이지 네비게이션** | 이전/다음 버튼으로 보도자료 간 쉬운 이동 |
| ⚠️ **결측치 처리** | 누락된 데이터를 모달창에서 직접 입력 |
| ✅ **검토 시스템** | 각 페이지 검토 완료 상태 추적 |
| ✨ **전체 생성** | 모든 페이지 검토 완료 후 일괄 생성 |
| 📊 **GRDP 데이터 연동** | 별도 GRDP 파일 업로드 지원 |

### 5.2 지원 보도자료 유형

#### 요약 보도자료 (9개)
- 표지, 일러두기, 목차, 인포그래픽
- 요약-지역경제동향, 요약-생산, 요약-소비건설, 요약-수출물가, 요약-고용인구

#### 부문별 보도자료 (10개)
- 광공업생산, 서비스업생산, 소비동향, 건설동향
- 수출, 수입, 물가동향
- 고용률, 실업률, 국내인구이동

#### 시도별 보도자료 (18개)
- 17개 시도별 개별 보도자료
- 참고_GRDP

#### 통계표 보도자료 (10개+)
- 통계표 목차
- 개별 통계표 (광공업생산지수, 서비스업생산지수, 소매판매액지수, 건설수주액, 고용률, 실업률, 국내인구이동, 수출액, 수입액, 소비자물가지수)
- 통계표-참고-GRDP, 부록-주요용어정의

> **총 약 50개 이상의 보도자료를 자동 생성**

## 8. 핵심 차별점

### 8.1 비공개 자료 처리 방식

#### ❌ 일반적인 접근

```
비공개 자료 존재
    ↓
"비공개 자료라서 구현할 수 없습니다"
    ↓
해당 기능 제외
    ↓
불완전한 시스템
```

#### ✅ 우리의 접근

```
비공개 자료 존재
    ↓
결측치 자동 감지
    ↓
실무자에게 모달창으로 입력 요청
    ↓
실무자가 직접 입력
    ↓
완전한 보도자료 생성
```

### 8.2 접근 방식 비교

| 구분 | 일반적인 접근 | 우리의 접근 |
|------|--------------|------------|
| **비공개 자료** | 핑계로 사용, 기능 제외 | 실무자 입력 방식으로 해결 |
| **시스템 완성도** | 불완전한 시스템 | 완전한 시스템 |
| **실무 적용** | 어려움 | 바로 사용 가능 |
| **사용자 경험** | 제한적 | 실무자 친화적 |

### 8.3 설계 철학

> **"비공개 자료는 문제가 아니라 해결 가능한 과제다"**
> 
> **"실무자의 전문성을 활용하여 문제를 해결한다"**
> 
> **"완벽한 자동화보다는 실무자와의 협업이 중요하다"**

## Q&A

### 자주 묻는 질문

<details>
<summary><strong>Q1. 이 시스템의 핵심 가치는 무엇인가요?</strong></summary>

**시간 절감과 업무 효율화**입니다. 초안 작성 시간을 80% 절감하고, 해석 작업 시간을 3배 확보하여 국민에게 더 의미 있는 분석을 제공할 수 있습니다.

</details>

<details>
<summary><strong>Q2. 비공개 자료 문제는 어떻게 해결했나요?</strong></summary>

결측치를 자동으로 감지하고, 실무자가 모달창을 통해 직접 입력하도록 설계했습니다. 이를 통해 비공개 자료 문제를 우회하면서도 완전한 시스템을 구현했습니다.

</details>

<details>
<summary><strong>Q3. 기존 시스템이나 다른 팀과의 차별점은?</strong></summary>

비공개 자료를 핑계로 대지 않고 실무자 입력 방식으로 해결한 것이 핵심 차별점입니다. 이를 통해 실무에서 바로 사용 가능한 완전한 시스템을 만들 수 있었습니다.

</details>

<details>
<summary><strong>Q4. 실제 생성에 걸리는 시간은?</strong></summary>

약 **3분** 내에 50개 이상의 보도자료가 자동 생성됩니다. 이후 편집 및 검토에 약 1일이 소요됩니다.

</details>

<details>
<summary><strong>Q5. 향후 확장 계획은?</strong></summary>

KOSIS API 연동을 통한 실시간 데이터 수집, AI 기반 해석 지원, 다국어 지원 등을 계획하고 있습니다.

</details>
