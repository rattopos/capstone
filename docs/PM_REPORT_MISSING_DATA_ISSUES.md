## 📋 Executive Summary

2025년 3분기 보도자료 HTML 파일에서 **서비스업생산, 수출, 수입, 물가** 보도자료에 대량의 결측치(N/A)가 발견되었습니다. 광공업생산은 정상적으로 표시되지만, 다른 보도자료들은 전국 데이터, 지역별 데이터, 테이블 데이터 등에서 결측치가 발생하고 있습니다.

**영향 범위**:
- 서비스업생산: 전국 지수, 지역별 지수, 지역별 세부업종 데이터
- 수출: 전국 수출액, 증감률, 주요 품목, 전체 테이블 데이터
- 수입: 전국 수입액, 증감률
- 물가: 전국 소비자물가, 품목 데이터

## 🔬 원인 분석: 모든 가능성 검토

### 가능성 1: 동적 컬럼 감지 실패

#### 문제 설명
엑셀 파일의 컬럼 구조가 예상과 다를 때, 동적 컬럼 감지 로직이 올바른 컬럼 인덱스를 찾지 못하는 경우입니다.

#### 기술적 세부사항
- **파일**: `utils/column_detector.py`
- **함수**: `get_column_mapping()`, `find_quarter_columns()`
- **문제점**:
  - 헤더 행 인덱스 자동 감지 실패
  - 분기별 컬럼 이름 패턴 매칭 실패 (예: "2025.2/4", "2025.3/4p" 등)
  - 기초자료 형식 vs 분석표 형식 구분 실패

#### 영향 범위
- **서비스업생산**: `production_index` 추출 실패 (집계 시트에서 지수 컬럼을 찾지 못함)
- **수출/수입**: 하드코딩된 컬럼 인덱스(22, 26) 사용으로 인한 데이터 추출 실패

#### 발생 가능성: ⭐⭐⭐⭐⭐ (매우 높음)

---

### 가능성 2: 하드코딩된 컬럼 인덱스 사용

#### 문제 설명
수출/수입 보도자료 생성 코드에서 동적 컬럼 감지를 사용하지 않고 하드코딩된 컬럼 인덱스를 사용하는 경우입니다.

#### 기술적 세부사항
- **파일**: `src/generators/export_generator.py`
- **문제 코드**:
  ```python
  change = row[22]  # 2025.2/4 증감률 (하드코딩)
  amount = row[26] / 10  # 2025.2/4 수출액 (하드코딩)
  ```
- **문제점**:
  - 엑셀 파일 구조가 변경되면 컬럼 인덱스가 달라짐
  - 기초자료 형식과 분석표 형식의 컬럼 인덱스가 다름
  - 분기별로 컬럼 위치가 달라질 수 있음

#### 영향 범위
- **수출**: 전국 수출액, 증감률, 테이블 데이터 전체
- **수입**: 전국 수입액, 증감률

#### 발생 가능성: ⭐⭐⭐⭐⭐ (매우 높음)

---

### 가능성 3: 집계 시트 데이터 추출 실패

#### 문제 설명
분석 시트가 비어있거나 데이터가 부족할 때 집계 시트에서 데이터를 추출하는 로직이 실패하는 경우입니다.

#### 기술적 세부사항
- **파일**: `src/generators/service_industry_generator.py`, `src/generators/export_generator.py`
- **함수**: `_get_nationwide_from_aggregation()`, `_get_sido_data_from_aggregation()`
- **문제점**:
  - 집계 시트에서 전국 행을 찾지 못함 (산업코드 'E~S' 또는 분류단계 0)
  - 분기별 컬럼 인덱스를 찾지 못함
  - 데이터 타입 변환 실패 (문자열, 숫자, None 등)

#### 영향 범위
- **서비스업생산**: 전국 지수 (`production_index = None`)
- **수출/수입**: 전국 데이터 전체

#### 발생 가능성: ⭐⭐⭐⭐ (높음)

---

### 가능성 4: 지역별 데이터 추출 실패

#### 문제 설명
지역별 데이터를 추출할 때 지역 인덱스를 찾지 못하거나, 해당 지역의 데이터 행을 찾지 못하는 경우입니다.

#### 기술적 세부사항
- **파일**: `src/generators/service_industry_generator.py`
- **함수**: `get_regional_data()`, `get_region_indices()`
- **문제점**:
  - 지역 인덱스 찾기 실패 (`get_region_indices()` 반환값이 비어있음)
  - 지역별 총지수 행 찾기 실패
  - 지역별 세부업종 데이터 추출 실패 (분류단계 1인 행을 찾지 못함)
  - 기여도 컬럼 찾기 실패

#### 영향 범위
- **서비스업생산**: 지역별 지수, 지역별 세부업종 데이터
- **수출/수입**: 지역별 데이터 전체

#### 발생 가능성: ⭐⭐⭐⭐ (높음)

---

### 가능성 5: 요약 박스 데이터 생성 실패

#### 문제 설명
요약 박스에 표시할 지역별 지수를 추출하는 로직이 누락되었거나 실패하는 경우입니다.

#### 기술적 세부사항
- **파일**: `src/generators/service_industry_generator.py`
- **함수**: `get_summary_box_data()`
- **문제점**:
  - 요약 박스 생성 시 지역별 지수(`index_2025`)를 추출하지 않음
  - `regional_data`에서 `index_2025` 값이 `None`인 경우 처리하지 않음
  - 템플릿에서 요약 박스 헤드라인에 지수를 표시하려고 하지만 데이터가 없음

#### 영향 범위
- **서비스업생산**: 요약 박스 헤드라인의 지역별 지수 (서울, 울산, 경기)

#### 발생 가능성: ⭐⭐⭐ (보통)

---

### 가능성 6: 데이터 타입 변환 실패

#### 문제 설명
엑셀에서 읽은 데이터를 숫자로 변환하는 과정에서 실패하거나, None 값이 제대로 처리되지 않는 경우입니다.

#### 기술적 세부사항
- **파일**: `src/generators/export_generator.py`, `src/generators/service_industry_generator.py`
- **함수**: `safe_float()`
- **문제점**:
  - 문자열로 저장된 숫자 변환 실패
  - 빈 셀, "-", "N/A" 등의 값 처리 실패
  - pandas의 `pd.isna()` 체크 실패

#### 영향 범위
- 모든 보도자료의 숫자 데이터

#### 발생 가능성: ⭐⭐ (낮음)

---

### 가능성 7: 시트 이름/구조 불일치

#### 문제 설명
엑셀 파일의 시트 이름이 예상과 다르거나, 시트 구조가 변경된 경우입니다.

#### 기술적 세부사항

- **문제점**:
  - 시트 이름 키워드 매칭 실패
  - 필수 컬럼/행 레이블 찾기 실패
  - 잘못된 시트를 선택함

#### 영향 범위
- 모든 보도자료 (특히 수출/수입)

#### 발생 가능성: ⭐⭐⭐ (보통)

### 가능성 8: 템플릿 호환성 레이어 문제

#### 문제 설명
데이터 생성기에서 생성한 데이터를 템플릿에 맞게 변환하는 호환성 레이어에서 데이터를 잘못 처리하거나 덮어쓰는 경우입니다.

#### 기술적 세부사항
- **파일**: `services/report_generator.py`
- **함수**: `_ensure_template_compatibility()`
- **문제점**:
  - `summary_box.main_increase_regions`를 덮어쓰면서 지역별 지수 정보 손실
  - None 값 처리 로직이 일관되지 않음
  - 데이터 구조 변환 중 정보 손실

#### 영향 범위
- 요약 박스 데이터
- 지역별 데이터

#### 발생 가능성: ⭐⭐ (낮음) - 최근 수정으로 대부분 해결됨

### 가능성 9: 분기 정보 누락

#### 문제 설명
데이터 추출 함수에 `year`와 `quarter` 파라미터가 전달되지 않아 올바른 분기 컬럼을 찾지 못하는 경우입니다.

#### 기술적 세부사항

- **함수**: `generate_report_data()`, `get_nationwide_data()`, `get_regional_data()`
- **문제점**:
  - `year=None`, `quarter=None`으로 호출되면 동적 컬럼 감지가 실패할 수 있음
  - 분기별 컬럼 이름 패턴 매칭 실패

#### 영향 범위
- 모든 보도자료 (특히 3분기 데이터)

#### 발생 가능성: ⭐⭐⭐ (보통)
