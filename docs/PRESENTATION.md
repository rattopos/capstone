## 1. 프로젝트 배경 및 목표

### 1.1 의뢰 기관 소개

**국가데이터처**는 국가 통계를 담당하는 핵심 기관으로, 66종의 승인통계를 작성하고 매월 20건 내외의 보도자료를 발표합니다.

### 1.2 제안 배경

국가데이터처에서 매 분기 발행하는 **"지역경제동향"** 보도자료는:

- 전국 17개 시도의 경제 지표를 종합 분석
- 10개 경제 부문(생산, 소비, 물가, 고용 등)을 다룸
- **약 78페이지** 분량으로 구성 (50개 보도자료 항목)
- 작성에 **전체 인력이 약 1주일** 투입

### 1.3 제안서 요구사항

> "AI를 이용한 통계자료 기반 보도자료 자동작성 시스템 개발"
> — 11기 캡스톤 프로젝트 제안서 (국가데이터처)

| 요구사항 | 설명 |
|---------|------|
| 데이터 전처리 | 엑셀 통계 자료 자동 로딩 |
| 통계표 자동 생성 | 데이터 기반 표 작성 |
| 그래프 자동 생성 | 시각화 차트 생성 |
| 설명 문구 자동 생성 | 통계 해설 문장 작성 |
| 템플릿 모듈화 | 재사용 가능한 구조 |

### 1.4 프로젝트 목표

> **분석표 엑셀 파일을 업로드하면 지역경제동향 보도자료를 자동으로 생성하는 시스템 구축**

## 2. 문제 분석

### 2.1 현재 업무 프로세스의 문제점

```
┌─────────────────────────────────────────────────────────────────┐
│                    현재 보도자료 작성 과정                        │
└─────────────────────────────────────────────────────────────────┘

[1일차] 데이터 수집 → [2~3일차] 분석표 작성 → [4~5일차] 보도자료 작성 → [6~7일차] 검토/수정

                                    ↑
                            여기가 반복 작업!
                            (복사-붙여넣기, 수치 입력, 문장 수정)
```

| 검토 및 수정 | 1~2일 | 오류 수정 |
| **총 소요 시간** | **약 1주일** | 전체 인력 투입 |

## 3. 해결 방안

### 3.1 접근 방식 비교

프로젝트 초기에 두 가지 접근 방식을 검토했습니다.

| 구분 | LLM(대규모 언어모델) | 규칙기반 시스템 |
|------|---------------------|----------------|
| **장점** | 자연스러운 문장, 유연한 대응 | 100% 정확도, 일관된 출력 |
| **단점** | 수치 오류 가능(hallucination), 결과 예측 불가 | 새로운 패턴 대응에 개발 필요 |
| **적합성** | 창의적 글쓰기, 요약 | **통계 보도자료** (정확성 필수) |

### 3.2 왜 규칙기반을 선택했는가?

**공식 보도자료의 특성:**

1. **정확성이 생명** — 통계 수치가 하나라도 틀리면 정정 보도 필요
2. **정형화된 패턴** — 매 분기 동일한 구조와 표현 사용
3. **검증 가능성** — 어떤 규칙으로 생성되었는지 추적 가능해야 함

> "0.1%의 오류도 허용되지 않는 공식 보도자료에서는 LLM의 hallucination 위험을 감수할 수 없습니다."

### 3.3 해결 전략

```
┌─────────────────────────────────────────────────────────────────┐
│                      자동화 전략                                  │
└─────────────────────────────────────────────────────────────────┘

   [분석표 엑셀]                    [규칙 기반 시스템]
        │                                │
        ▼                                ▼
   검증된 통계 데이터    +    사전 정의된 표현 규칙    
        │                                │
        └────────────────┬───────────────┘
                         │
                         ▼
                  [자동 생성된 보도자료]
                         │
                         ▼
                   정확하고 일관된 출력
```

## 5. 핵심 기술: 규칙기반 자연어 생성

> 💡 **중요한 발견**: 실제 보도자료 작성을 분석한 결과, **일관된 문장 생성 규칙**이 존재했습니다.  
> 이 규칙은 담당자들이 수년간 보도자료를 작성하면서 자연스럽게 형성된 **암묵적 표준**이었고,  
> 우리 시스템은 이 규칙을 **명시적으로 코드화**하여 자동 생성에 활용했습니다.

### 5.1 규칙기반 자연어 생성이란?

**사전에 정의된 규칙**에 따라 데이터를 문장으로 변환하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                   규칙기반 자연어 생성 원리                        │
└─────────────────────────────────────────────────────────────────┘

     [원본 데이터]              [규칙 적용]              [결과 문장]
         │                         │                       │
    증감률: 3.5%     ───▶     증가/감소 판단     ───▶    "3.5% 증가"
    지역: 경기       ───▶     지역명 매핑       ───▶    "경기"
    순위: 1위        ───▶     순위 표현 선택    ───▶    "가장 높았다"
```

### 5.2 적용된 6가지 규칙 유형

#### 규칙 1: 매핑 규칙 (이름 변환)

엑셀의 긴 공식 명칭을 보도자료용 간결한 표현으로 변환합니다.

| 엑셀 원본 (31자) | 보도자료 표기 (7자) |
|----------------|-------------------|
| 전자 부품, 컴퓨터, 영상, 음향 및 통신장비 제조업 | 반도체·전자부품 |
| 의료, 정밀, 광학 기기 및 시계 제조업 | 의료·정밀 |
| 자동차 및 트레일러 제조업 | 자동차 |

> 총 28개 업종, 17개 지역에 대한 매핑 규칙 정의

#### 규칙 2: 계산 규칙 (수치 처리)

| 규칙 | 설명 | 예시 |
|------|------|------|
| 소수점 처리 | 첫째 자리까지 반올림 | 3.456 → 3.5 |
| 백분율 변환 | 소수를 퍼센트로 | 0.035 → 3.5% |
| 천 단위 구분 | 쉼표 추가 | 1234567 → 1,234,567 |

#### 규칙 3: 정렬/분류 규칙

데이터를 의미 있는 순서로 정렬하고 그룹화합니다.

| 분류 기준 | 설명 |
|----------|------|
| 증감률 순 | 높은 순서대로 나열 |
| 지역 그룹 | 수도권/비수도권 구분 |
| 상위 N개 | 1~3위 업종만 추출 |

#### 규칙 4: 조건부 텍스트 생성

수치의 부호나 크기에 따라 다른 표현을 사용합니다.

| 조건 | 생성되는 텍스트 |
|------|----------------|
| 증감률 > 0 | "X.X% **증가**하였다" |
| 증감률 < 0 | "X.X% **감소**하였다" |
| 증감률 = 0 | "전분기와 **동일**하였다" |

추가 예시:
| 조건 | 생성되는 텍스트 |
|------|----------------|
| 1위 업종 | "**가장 높은** 증가율을 보였다" |
| 전국 평균 이상 | "전국 평균을 **상회**하였다" |
| 전국 평균 이하 | "전국 평균을 **하회**하였다" |

#### 규칙 4-1: 방향성 대조 표현 (핵심 규칙)

> ⚠️ **이 규칙은 기존 보도자료에서 일관되게 사용되던 암묵적 규칙이었습니다.**  
> 예: "석유화학에서 줄었으나, 반도체에서 늘어 3.2% 증가"  
> 우리 시스템은 이런 규칙을 발견하고 코드화하여 자동 생성에 적용했습니다.

보도자료에서 자주 사용되는 **"~하였으나, ~하여"** 패턴은 **최종 결과의 방향**에 따라 반대 표현을 먼저 배치합니다.

**핵심 원칙: 결과와 반대되는 요인을 먼저 언급하고, 결과와 같은 방향의 요인을 나중에 언급**

```
┌─────────────────────────────────────────────────────────────────┐
│              방향성 대조 표현 생성 원리                            │
└─────────────────────────────────────────────────────────────────┘

   [최종 결과가 "증가"인 경우]
   
   "반도체(3.2%)에서 줄었으나, 자동차(5.1%)에서 늘어 전년동분기대비 2.3% 증가"
         ↑                        ↑                           ↑
    반대 요인 먼저          결과 방향 요인           최종 결과
    (감소 업종)             (증가 업종)              (증가)


   [최종 결과가 "감소"인 경우]
   
   "자동차(5.1%)에서 늘었으나, 반도체(-3.2%)에서 줄어 전년동분기대비 1.5% 감소"
         ↑                        ↑                           ↑
    반대 요인 먼저          결과 방향 요인           최종 결과
    (증가 업종)             (감소 업종)              (감소)
```

**지표별 동사-결과 매칭 규칙:**

| 지표 | 결과가 양수일 때 | 결과가 음수일 때 |
|------|----------------|----------------|
| **생산/소비/수출입** | "줄었으나...늘어...증가" | "늘었으나...줄어...감소" |
| **물가/고용률** | "하락하였으나...올라...상승" | "상승하였으나...내려...하락" |
| **인구이동** | "유출되었으나...유입되어...순유입" | "유입되었으나...유출되어...순유출" |
| **물가 비교** | "낮았으나...높음" | "높았으나...낮음" |

> 💡 **"올라-내려" 패턴**: 물가와 고용률 지표에서는 다른 지표와 달리 **"올라/내려"** 동사를 사용합니다.
> - **"올라"** → "상승" 결과와 연결 (예: "등이 올라 전년동분기대비 상승")
> - **"내려"** → "하락" 결과와 연결 (예: "등이 내려 전년동분기대비 하락")
> - 전국 단위 나레이션에서도 일관되게 사용 (예: "전국 고용률은 ... 등이 올라 0.2%p 상승")

**실제 적용 예시:**

```
┌─────────────────────────────────────────────────────────────────┐
│     예시 1: 광공업 생산 (결과: 3.5% 증가)                          │
└─────────────────────────────────────────────────────────────────┘

   원본 데이터 (분석표):
   ┌────────────────┬───────────┐
   │     업종       │   증감률   │
   ├────────────────┼───────────┤
   │ 반도체·전자부품 │   +12.3%  │ ← 증가 업종
   │ 자동차         │   +7.8%   │ ← 증가 업종
   │ 석유화학       │   -5.2%   │ ← 감소 업종
   │ 철강           │   -3.1%   │ ← 감소 업종
   └────────────────┴───────────┘
   
   처리 과정:
   1. 전체 결과 판단: 3.5% > 0 → "증가"
   2. 증가 업종 추출: 반도체·전자부품, 자동차
   3. 감소 업종 추출: 석유화학, 철강
   4. 결과가 "증가"이므로 감소 업종 먼저, 증가 업종 나중에 배치
   
   생성 결과:
   "석유화학(-5.2%), 철강(-3.1%) 등에서 줄었으나, 
    반도체·전자부품(12.3%), 자동차(7.8%) 등에서 늘어 
    전년동분기대비 3.5% 증가"
```


#### 규칙 4-2: 분류 단계별 대조 패턴 (증-증-감-증, 감-감-증-감)

> ⚠️ **이 규칙도 기존 보도자료에서 발견된 중요한 패턴입니다.**  
> 수출/수입 등 일부 지표에서는 **전국-시도-품목** 3단계 분류에서 각각의 증감 방향을 조합하여 나레이션을 생성합니다.

보도자료에서 수출/수입 등 일부 지표는 **3단계 분류(전국 → 시도 → 품목)**에서 각 단계의 증감 방향을 조합하여 나레이션을 생성합니다.

**핵심 원칙: 전국 레벨의 결과 방향에 따라 시도와 품목의 대조 패턴 결정**

```
┌─────────────────────────────────────────────────────────────────┐
│          분류 단계별 대조 패턴 구조                               │
└─────────────────────────────────────────────────────────────────┘

  [단계 1: 전국 레벨]
  └─ 전국 증감률 (예: +2.1% 증가)
     │
     ├─ [단계 2: 시도 레벨]
     │  ├─ 증가 시도 (예: 제주, 충북, 경남 등 10개)
     │  └─ 감소 시도 (예: 세종, 전남, 부산 등 7개)
     │     │
     │     └─ [단계 3: 품목 레벨]
     │        ├─ 증가 시도의 품목 (예: 프로세서·컨트롤러, 메모리 반도체)
     │        └─ 감소 시도의 품목 (예: 기타 인조플라스틱)
```

**패턴 유형:**

| 패턴 코드 | 전국 | 시도 (다수) | 품목 (대조) | 최종 결과 | 설명 |
|----------|------|------------|------------|----------|------|
| **증-증-감-증** | 증가 | 증가 시도 많음 | 감소 품목 있음 | 증가 | 전국 증가, 대부분 시도 증가, 일부 품목은 감소하지만 전체 증가 |
| **감-감-증-감** | 감소 | 감소 시도 많음 | 증가 품목 있음 | 감소 | 전국 감소, 대부분 시도 감소, 일부 품목은 증가하지만 전체 감소 |

**실제 적용 예시:**

```
┌─────────────────────────────────────────────────────────────────┐
│     예시 1: 수출 (증-증-감-증 패턴)                               │
└─────────────────────────────────────────────────────────────────┘

   분류 단계별 데이터:
   
   [전국 레벨] 전년동분기대비 2.1% 증가
   
   [시도 레벨]
   - 증가 시도: 제주(+37.8%), 충북(+34.9%), 경남(+12.9%) 등 10개
   - 감소 시도: 세종(-37.2%), 전남(-13.7%), 부산(-6.0%) 등 7개
   
   [품목 레벨]
   - 증가 시도의 품목: 프로세서·컨트롤러, 메모리 반도체, 기타 중화학 공업품
   - 감소 시도의 품목: 기타 인조플라스틱 및 동 제품, 선박
   
   생성된 나레이션:
   
   ① nationwide_summary (전국-품목 레벨):
      "전국 수출(17,515.8억달러)은 메모리 반도체, 선박, 기타 중화학 공업품 
       등의 수출이 늘어 전년동분기대비 2.1% 증가"
      ↑
      [전국: 증가] + [품목: 증가]
   
   ② regional_summary (시도-품목 레벨, 전국 증가이므로 감소→증가 순서):
      "세종(-37.2%), 전남(-13.7%), 부산(-6.0%)은 기타 인조플라스틱 등의 
       수출이 줄어 감소하였으나, 
       제주(37.8%), 충북(34.9%), 경남(12.9%)은 프로세서·컨트롤러 등의 
       수출이 늘어 증가"
      ↑
      [시도: 감소] + [품목: 감소] → [시도: 증가] + [품목: 증가]
```

```
┌─────────────────────────────────────────────────────────────────┐
│     예시 2: 수입 (감-감-증-감 패턴)                               │
└─────────────────────────────────────────────────────────────────┘

   분류 단계별 데이터:
   
   [전국 레벨] 전년동분기대비 1.7% 감소
   
   [시도 레벨]
   - 감소 시도: 충남(-16.7%), 전남(-13.4%), 경북(-13.1%) 등 11개
   - 증가 시도: 제주(+53.7%), 광주(+27.7%), 세종(+8.1%) 등 6개
   
   [품목 레벨]
   - 감소 시도의 품목: 원유, 기타 무기화합물
   - 증가 시도의 품목: 항공기, 프로세서·컨트롤러, 컴퓨터 주변기기
   
   생성된 나레이션:
   
   ① nationwide_summary (전국-품목 레벨):
      "전국 수입(15,433.2억달러)은 원유, 석탄, 나프타 등의 수입이 줄어 
       전년동분기대비 1.7% 감소"
      ↑
      [전국: 감소] + [품목: 감소]
   
   ② regional_summary (시도-품목 레벨, 전국 감소이므로 증가→감소 순서):
      "제주(53.7%), 광주(27.7%), 세종(8.1%)은 항공기, 프로세서·컨트롤러 
       등의 수입이 늘어 증가하였으나, 
       충남(-16.7%), 전남(-13.4%), 경북(-13.1%)은 원유, 기타 무기화합물 
       등의 수입이 줄어 감소"
      ↑
      [시도: 증가] + [품목: 증가] → [시도: 감소] + [품목: 감소]
```

**패턴 적용 규칙 요약:**

1. **전국 레벨(nationwide_summary)**:
   - 전국 증감률과 전국 상위 품목을 결합
   - 전국 증가 → "늘어 증가"
   - 전국 감소 → "줄어 감소"

2. **시도-품목 레벨(regional_summary)**:
   - 전국 결과 방향에 따라 대조 패턴 결정
   - 전국 증가 → "감소 시도(감소 품목) → 증가 시도(증가 품목)"
   - 전국 감소 → "증가 시도(증가 품목) → 감소 시도(감소 품목)"

3. **품목 선택 기준**:
   - 각 시도별 상위 순위 품목 선택 (기여도 순)
   - 증가 시도: 증가 품목 선택
   - 감소 시도: 감소 품목 선택

**이 패턴의 의미:**

이 규칙은 통계 해석에서 **다층적 분석 구조**를 보여줍니다:
- 전체 추세와 지역별/품목별 차이를 동시에 설명
- 전국 단위의 결과가 지역/품목 단위에서는 다양할 수 있음을 명확히 표현
- 보도자료 독자가 **전체 그림과 세부 사항을 동시에 이해**할 수 있도록 도움

#### 규칙 4-3: 지표별 동사 선택 규칙 (올라-내려 vs 늘어-줄어)

> ⚠️ **이 규칙도 기존 보도자료에서 발견된 중요한 패턴입니다.**  
> 지표의 성격에 따라 다른 동사를 사용하여 자연스러운 표현을 만듭니다.

보도자료에서는 지표의 성격에 따라 **서로 다른 동사**를 사용합니다:

**동사 사용 규칙:**

| 지표 유형 | 증가 동사 | 감소 동사 | 최종 결과 표현 |
|----------|---------|---------|--------------|
| **생산/소비/수출입** | 늘어 | 줄어 | 증가/감소 |
| **물가/고용률** | 올라 | 내려 | 상승/하락 |

**실제 적용 예시:**

```
┌─────────────────────────────────────────────────────────────────┐
│     생산 지표 (늘어-줄어 패턴)                                    │
└─────────────────────────────────────────────────────────────────┘

  "전국 광공업생산(108.2)은 반도체·전자부품(12.3%), 자동차(7.8%) 등의 
   생산이 늘어 전년동분기대비 3.2% 증가"
   
  💡 "늘어" → "증가" 결과와 연결


┌─────────────────────────────────────────────────────────────────┐
│     물가 지표 (올라-내려 패턴)                                    │
└─────────────────────────────────────────────────────────────────┘

  "전국 소비자물가(116.3)는 외식제외개인서비스, 외식 등이 올라 
   전년동분기대비 2.1% 상승"
   
  💡 "올라" → "상승" 결과와 연결


┌─────────────────────────────────────────────────────────────────┐
│     고용률 지표 (올라-내려 패턴)                                  │
└─────────────────────────────────────────────────────────────────┘

  "전국 고용률은 63.5%로, 30~39세, 40~49세의 고용률이 올라 
   전년동분기대비 0.2%p 상승"
   
  💡 "올라" → "상승" 결과와 연결
```

**스키마에서의 정의:**

```json
{
  "nationwide_trend": {
    "increase": {
      "expression": "올라",    // 물가/고용률
      "result": "상승"
    },
    "decrease": {
      "expression": "내려",    // 물가/고용률
      "result": "하락"
    }
  }
}
```

vs

```json
{
  "nationwide_trend": {
    "increase": {
      "expression": "늘어",    // 생산/소비/수출입
      "result": "증가"
    },
    "decrease": {
      "expression": "줄어",    // 생산/소비/수출입
      "result": "감소"
    }
  }
}
```

**이 규칙의 의미:**

이 규칙은 보도자료의 **자연스러운 표현**을 위해 중요합니다:
- "물가가 올라/내려" → 물가의 상승/하락을 직관적으로 표현
- "생산이 늘어/줄어" → 생산량의 증가/감소를 명확하게 표현
- 지표의 성격에 맞는 동사 선택으로 **읽기 쉬운 문장** 생성

**템플릿(HTML)에서의 구현:**

```jinja2

    {% if emp_inc | length > 0 %}{{ emp_inc }}에서 상승하였으나, {% endif %}
    {% if emp_dec | length > 0 %}{{ emp_dec }}에서 내려 {% endif %}
    전년동분기대비 {{ rate }}%p 하락
{% endif %}
```

### 5.3 원데이터에서 나레이션 생성 전체 과정

실제 엑셀 원데이터가 어떻게 보도자료 나레이션으로 변환되는지 전체 과정을 상세히 설명합니다.

```
┌─────────────────────────────────────────────────────────────────┐
│          원데이터 → 나레이션 생성 7단계 프로세스                    │
└─────────────────────────────────────────────────────────────────┘

  [1] 엑셀 파싱      [2] 데이터 추출      [3] 분류/정렬
       │                  │                   │
       ▼                  ▼                   ▼
  시트명 찾기       셀 위치에서        증가/감소 분류
  헤더 식별         값 읽기           기여도순 정렬
       │                  │                   │
       └──────────────────┴───────────────────┘
                          │
                          ▼
                    [4] 방향성 판단
                    (증가/감소/상승/하락)
                          │
                          ▼
                    [5] 매핑 적용
                    (업종명, 지역명 변환)
                          │
                          ▼
                    [6] 템플릿 선택
```

#### 8.2.2 배포 후 테스트

1. **실무자 테스트 기간**
   - 배포 후 실무자들의 실제 업무 환경에서 테스트
   - 피드백 수집 및 개선 사항 반영

2. **사용자 만족도 기대**
   - 초안을 빠르게 생성하여 한글 파일로 붙여넣기만 하면 됨
   - 편집 및 검토 작업만 수행하면 되므로 **단순 반복 작업에서 해방**
   - 실수에 대한 긴장에서 벗어나 **업무 피로도 및 시간 비효율성 해소**
   - **사용자 만족도가 매우 높을 것으로 기대**

#### 8.2.3 배포 형태의 장점

**망분리 환경 대응:**
- 국가데이터처는 망분리 환경으로 일반적인 배포 방식 어려움
- PyInstaller로 단일 실행 파일 생성 → USB로 전달 가능
- 더블클릭으로 즉시 실행, 별도 설치 불필요
- 의존성 패키지 모두 포함

### 8.3 향후 발전 방향

#### 단기 (1~3개월)
- 실무자 피드백 반영 및 버그 수정
- HWP 직접 출력 기능 (현재는 HTML 복사-붙여넣기)
- 데이터 검증 자동화 강화

#### 중기 (3~6개월)
- 차트 이미지 자동 생성
- 승인 워크플로우 구현

#### 장기 (6개월~1년)
- 다른 보도자료로 확장 (인구동향, 물가동향 등)
- AI 기반 분석 코멘트 생성 (LLM 도입 검토)

### 8.4 확장 가능성

본 시스템의 구조(규칙기반 + 템플릿)는 다른 보도자료에도 적용 가능합니다:

| 확장 대상 | 적용 방법 |
|----------|----------|
| 인구동향 보도자료 | 새 템플릿 + 생성기 추가 |
| 물가동향 보도자료 | 새 템플릿 + 생성기 추가 |
| 고용동향 보도자료 | 새 템플릿 + 생성기 추가 |

> 제안서 최종 목표: "타 통계에도 적용 가능한 AI 활용 보도자료 생성 모델 개발"

## 9. Q&A 예상 질문

### 질문 1: 왜 LLM을 사용하지 않았나요?

**답변:**

공식 통계 보도자료는 **0.1%의 오류도 허용되지 않습니다.**

| 구분 | LLM의 위험성 | 규칙기반의 안전성 |
|------|-------------|------------------|
| 수치 | 가끔 잘못된 숫자 생성 가능 | 원본 그대로 사용 |
| 표현 | 매번 다른 문장 생성 | 정해진 패턴 사용 |
| 검증 | 결과 예측 불가 | 규칙으로 추적 가능 |

향후 고도화 단계에서 **분석 코멘트 생성** 등 제한된 영역에 LLM 도입을 검토하고 있습니다.

---

### 질문 2: 제안서에서 요구한 KOSIS API 연동은 어떻게 됐나요?

**답변:**

프로젝트 초기에는 KOSIS API에서 직접 데이터를 수집하는 방식을 검토했으나, 업무 분석 과정에서 **더 효율적인 방법**을 발견했습니다.

| 접근 방식 | Bottom-up (KOSIS API) | Top-down (분석표) |
|----------|----------------------|-------------------|
| 데이터 소스 | 원시 데이터 | **이미 검증된 데이터** |
| 품질 보증 | 추가 검증 필요 | 담당자가 이미 검토 완료 |
| 구현 복잡도 | API 연동 + 집계 로직 필요 | 엑셀 파싱만 필요 |

국가데이터처에서는 이미 KOSIS 데이터를 집계하여 **분석표**를 만들고 있었습니다. 이 검증된 데이터를 활용하는 것이 더 안전하고 효율적입니다.

---

### 질문 3: HWP 파일로 직접 출력할 수 없나요?

**답변:**

현재는 **HTML → 한글 복사-붙여넣기** 방식을 사용합니다.

| 방식 | 설명 |
|------|------|
| **현재** | HTML 생성 → 한글에 복사-붙여넣기 |
| **향후** | python-hwp 라이브러리로 직접 HWP 생성 예정 |

복사-붙여넣기 방식도 **표와 서식이 그대로 유지**되어 실무에서 충분히 활용 가능합니다.

---

### 질문 4: 데이터가 누락되면 어떻게 되나요?

**답변:**

시스템은 **안전하게 대응**합니다:

| 상황 | 처리 방법 |
|------|----------|
| 값이 없음 | `[ ]` 빈 칸으로 표시 |
| 형식 오류 | 원본 값 그대로 출력 |
| 시트 누락 | 해당 보도자료 생성 건너뜀 |

담당자가 **육안으로 확인하고 직접 입력**할 수 있도록 빈 칸으로 표시합니다. 잘못된 값을 자동으로 만들어내는 것보다 안전합니다.

---

### 질문 5: 새로운 보도자료를 추가하려면?

**답변:**

시스템이 **모듈화되어 있어** 추가가 용이합니다:

```
새 보도자료 추가 절차:

1. 템플릿 작성 (HTML 서식)
2. 생성기 작성 (엑셀에서 데이터 추출)
3. 설정에 등록

→ 기존 시스템 수정 없이 확장 가능
```

이것이 **확장 가능한 프레임워크**로 설계한 이유입니다.

---

### 질문 6: 테스트는 어떻게 했나요?

**답변:**

| 테스트 방법 | 내용 |
|------------|------|
| **정답 비교** | 기존 보도자료(정답)와 생성 결과 비교 |
| **수치 검증** | 엑셀 원본과 생성된 수치 대조 |
| **육안 검토** | 담당자 피드백 반영 |

`correct_answer/` 폴더에 기존 보도자료 이미지를 저장하여 비교 검토했습니다.

---

### 질문 7: 배포는 어떻게 하나요?

**답변:**

국가데이터처는 **망분리 환경**으로 일반적인 방법이 어렵습니다.

| 방식 | 가능 여부 |
|------|----------|
| 클라우드 배포 | ❌ 불가 (망분리) |
| pip 설치 | ❌ 어려움 (인터넷 차단) |
| **실행 파일(.exe)** | ✅ **최적** |

PyInstaller로 **단일 실행 파일**을 만들어 USB로 전달하면:
- 더블클릭으로 즉시 실행
- 별도 설치 불필요
- 의존성 패키지 포함

## 부록: 스키마 방식 vs 하드코딩 방식

본 프로젝트는 **스키마 기반**과 **하드코딩** 두 가지 방식이 혼재되어 있습니다.

### 스키마 방식 (유연함) ✅

| 구분 | 파일 | 설명 |
|------|------|------|
| 데이터 구조 정의 | `*_schema.json` (21개) | 보도자료별 데이터 필드 정의 |
| 업종명 매핑 | 스키마 내 `industry_name_mapping` | 엑셀 업종명 → 보도자료 표기명 |
| 텍스트 생성 규칙 | 스키마 내 `rules` | 증가/감소 문장 패턴 |

### 하드코딩 방식 (Python 수정 필요) ⚠️

| 구분 | 위치 | 문제점 |
|------|------|--------|
| 시트명 | Generator 클래스 | 시트명 변경 시 코드 수정 |
| 컬럼 인덱스 | Generator 메서드 | `region_agg[18]`, `[22]` 등 |
| 지역 목록 | Generator 클래스 상수 | 지역 추가 시 코드 수정 |
| 보도자료 목록 | `config/reports.py` | 보도자료 추가 시 코드 수정 |

### 현재 상태

| 구분 | 스키마 | 하드코딩 |
|------|--------|----------|
| 데이터 구조 | ✅ 100% | - |
| 텍스트 규칙 | ✅ 100% | - |
| 컬럼 인덱스 | ⚠️ 10% | ⚠️ 90% |
| 시트명 | - | ⚠️ 100% |

**종합**: 약 **40% 스키마**, **60% 하드코딩**

> 💡 향후 개선: 컬럼 인덱스와 시트명을 스키마로 이전하면 엑셀 형식 변경에 코드 수정 없이 대응 가능
