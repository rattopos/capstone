## 1. 프로젝트 배경 및 목표

### 1.1 의뢰 기관 소개

**국가데이터처**는 국가 통계를 담당하는 핵심 기관으로, 66종의 승인통계를 작성하고 매월 20건 내외의 보도자료를 발표합니다.

### 1.2 제안 배경

국가데이터처에서 매 분기 발행하는 **"지역경제동향"** 보도자료는:

- 전국 17개 시도의 경제 지표를 종합 분석
- 10개 경제 부문(생산, 소비, 물가, 고용 등)을 다룸
- **50페이지 이상**의 분량으로 구성
- 작성에 **전체 인력이 약 1주일** 투입

### 1.3 제안서 요구사항

> "AI를 이용한 통계자료 기반 보도자료 자동작성 시스템 개발"
> — 11기 캡스톤 프로젝트 제안서 (국가데이터처)

| 요구사항 | 설명 |
|---------|------|
| 데이터 전처리 | 엑셀 통계 자료 자동 로딩 |
| 통계표 자동 생성 | 데이터 기반 표 작성 |
| 그래프 자동 생성 | 시각화 차트 생성 |
| 설명 문구 자동 생성 | 통계 해설 문장 작성 |
| 템플릿 모듈화 | 재사용 가능한 구조 |

### 1.4 프로젝트 목표

> **분석표 엑셀 파일을 업로드하면 지역경제동향 보도자료를 자동으로 생성하는 시스템 구축**

## 2. 문제 분석

### 2.1 현재 업무 프로세스의 문제점

```
┌─────────────────────────────────────────────────────────────────┐
│                    현재 보도자료 작성 과정                        │
└─────────────────────────────────────────────────────────────────┘

[1일차] 데이터 수집 → [2~3일차] 분석표 작성 → [4~5일차] 보도자료 작성 → [6~7일차] 검토/수정

                                    ↑
                            여기가 반복 작업!
                            (복사-붙여넣기, 수치 입력, 문장 수정)
```

### 2.2 핵심 문제

| 문제 | 설명 |
|------|------|
| **반복 작업** | 동일한 패턴의 표, 그래프, 문장을 매번 수작업으로 작성 |
| **휴먼 에러** | 수치 오입력, 복사 실수, 서식 불일치 발생 |
| **인력 낭비** | 숙련된 통계 전문가가 단순 반복 작업에 시간 소비 |
| **일관성 부족** | 담당자마다 다른 표현과 서식 사용 |

### 2.3 작업 시간 분석

| 작업 단계 | 현재 소요 시간 | 비고 |
|----------|---------------|------|
| 데이터 수집/정리 | 2~3일 | 엑셀 작업 |
| 보도자료 초안 작성 | 2~3일 | **반복 작업** |
| 검토 및 수정 | 1~2일 | 오류 수정 |
| **총 소요 시간** | **약 1주일** | 전체 인력 투입 |

## 3. 해결 방안

### 3.1 접근 방식 비교

프로젝트 초기에 두 가지 접근 방식을 검토했습니다.

| 구분 | LLM(대규모 언어모델) | 규칙기반 시스템 |
|------|---------------------|----------------|
| **장점** | 자연스러운 문장, 유연한 대응 | 100% 정확도, 일관된 출력 |
| **단점** | 수치 오류 가능(hallucination), 결과 예측 불가 | 새로운 패턴 대응에 개발 필요 |
| **적합성** | 창의적 글쓰기, 요약 | **통계 보도자료** (정확성 필수) |

### 3.2 왜 규칙기반을 선택했는가?

**공식 보도자료의 특성:**

1. **정확성이 생명** — 통계 수치가 하나라도 틀리면 정정 보도 필요
2. **정형화된 패턴** — 매 분기 동일한 구조와 표현 사용
3. **검증 가능성** — 어떤 규칙으로 생성되었는지 추적 가능해야 함

> "0.1%의 오류도 허용되지 않는 공식 보도자료에서는 LLM의 hallucination 위험을 감수할 수 없습니다."

### 3.3 해결 전략

```
┌─────────────────────────────────────────────────────────────────┐
│                      자동화 전략                                  │
└─────────────────────────────────────────────────────────────────┘

   [분석표 엑셀]                    [규칙 기반 시스템]
        │                                │
        ▼                                ▼
   검증된 통계 데이터    +    사전 정의된 표현 규칙    
        │                                │
        └────────────────┬───────────────┘
                         │
                         ▼
                  [자동 생성된 보도자료]
                         │
                         ▼
                   정확하고 일관된 출력
```

## 4. 시스템 소개

### 4.1 시스템 개요

웹 브라우저에서 동작하는 **대시보드 형태**의 보도자료 자동 생성 시스템입니다.

### 4.2 사용 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│                     시스템 사용 흐름                              │
└─────────────────────────────────────────────────────────────────┘

  ① 파일 업로드          ② 미리보기           ③ 내보내기
  ┌──────────┐         ┌──────────┐         ┌──────────┐
  │          │         │          │         │          │
  │  분석표   │  ───▶  │  미리보기  │  ───▶  │   HTML   │
  │  엑셀    │         │  및 수정  │         │  다운로드 │
  │          │         │          │         │          │
  └──────────┘         └──────────┘         └──────────┘
     드래그 앤 드롭         실시간 확인          한글 복사용
```

### 4.3 대시보드 구성

| 탭 | 내용 | 페이지 수 |
|----|------|----------|
| **요약** | 표지, 목차, 인포그래픽, 요약문 | 9개 |
| **부문별** | 광공업, 서비스업, 소비, 건설, 수출입, 물가, 고용, 인구이동 | 10개 |
| **시도별** | 서울~제주 17개 시도 + GRDP 참고 | 18개 |
| **통계표** | 각 지표별 통계표, 부록 | 13개 |
| **합계** | | **50개 페이지** |

### 4.4 주요 기능

| 기능 | 설명 |
|------|------|
| **자동 연도/분기 감지** | 엑셀 파일명에서 자동으로 "2025년 2분기" 추출 |
| **실시간 미리보기** | 클릭 한 번으로 즉시 보도자료 미리보기 |
| **일괄 생성** | 모든 50개 페이지 한 번에 생성 |
| **수정 가능** | 생성된 내용에서 직접 편집 가능 |
| **내보내기** | HTML로 저장 후 한글(HWP)에 복사-붙여넣기 |

## 5. 핵심 기술: 규칙기반 자연어 생성

### 5.1 규칙기반 자연어 생성이란?

**사전에 정의된 규칙**에 따라 데이터를 문장으로 변환하는 기술입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                   규칙기반 자연어 생성 원리                        │
└─────────────────────────────────────────────────────────────────┘

     [원본 데이터]              [규칙 적용]              [결과 문장]
         │                         │                       │
    증감률: 3.5%     ───▶     증가/감소 판단     ───▶    "3.5% 증가"
    지역: 경기       ───▶     지역명 매핑       ───▶    "경기"
    순위: 1위        ───▶     순위 표현 선택    ───▶    "가장 높았다"
```

### 5.2 적용된 6가지 규칙 유형

#### 규칙 1: 매핑 규칙 (이름 변환)

엑셀의 긴 공식 명칭을 보도자료용 간결한 표현으로 변환합니다.

| 엑셀 원본 (31자) | 보도자료 표기 (7자) |
|----------------|-------------------|
| 전자 부품, 컴퓨터, 영상, 음향 및 통신장비 제조업 | 반도체·전자부품 |
| 의료, 정밀, 광학 기기 및 시계 제조업 | 의료·정밀 |
| 자동차 및 트레일러 제조업 | 자동차 |

> 총 28개 업종, 17개 지역에 대한 매핑 규칙 정의

#### 규칙 2: 계산 규칙 (수치 처리)

| 규칙 | 설명 | 예시 |
|------|------|------|
| 소수점 처리 | 첫째 자리까지 반올림 | 3.456 → 3.5 |
| 백분율 변환 | 소수를 퍼센트로 | 0.035 → 3.5% |
| 천 단위 구분 | 쉼표 추가 | 1234567 → 1,234,567 |

#### 규칙 3: 정렬/분류 규칙

데이터를 의미 있는 순서로 정렬하고 그룹화합니다.

| 분류 기준 | 설명 |
|----------|------|
| 증감률 순 | 높은 순서대로 나열 |
| 지역 그룹 | 수도권/비수도권 구분 |
| 상위 N개 | 1~3위 업종만 추출 |

#### 규칙 4: 조건부 텍스트 생성

수치의 부호나 크기에 따라 다른 표현을 사용합니다.

| 조건 | 생성되는 텍스트 |
|------|----------------|
| 증감률 > 0 | "X.X% **증가**하였다" |
| 증감률 < 0 | "X.X% **감소**하였다" |
| 증감률 = 0 | "전분기와 **동일**하였다" |

추가 예시:
| 조건 | 생성되는 텍스트 |
|------|----------------|
| 1위 업종 | "**가장 높은** 증가율을 보였다" |
| 전국 평균 이상 | "전국 평균을 **상회**하였다" |
| 전국 평균 이하 | "전국 평균을 **하회**하였다" |

#### 규칙 5: 문장 템플릿

정해진 문장 구조에 데이터를 삽입합니다.

```
템플릿: "{지역}의 {지표}은(는) 전분기 대비 {증감률}% {증감표현}하였다."

  ↓ 데이터 삽입

결과: "경기의 광공업생산은 전분기 대비 3.5% 증가하였다."
```

#### 규칙 6: 결측치 처리

데이터가 없는 경우 안전하게 처리합니다.

| 상황 | 처리 방법 |
|------|----------|
| 값이 없음 | "[ ]" 플레이스홀더 표시 |
| 수식 오류 | 기본값 또는 경고 메시지 |
| 형식 오류 | 원본 값 그대로 표시 |

> 담당자가 직접 확인하고 수정할 수 있도록 빈 칸으로 표시

### 5.3 왜 이 방식이 효과적인가?

| 특성 | 규칙기반의 장점 |
|------|----------------|
| **정확성** | 정의된 규칙대로만 동작 → 오류 없음 |
| **일관성** | 동일한 입력 → 항상 동일한 출력 |
| **추적성** | 왜 이런 문장이 나왔는지 규칙으로 설명 가능 |
| **수정 용이** | 규칙만 수정하면 전체 출력에 반영 |

### 5.4 LLM과의 비교

| 구분 | LLM | 규칙기반 (본 시스템) |
|------|-----|---------------------|
| 수치 정확도 | 가끔 오류 발생 가능 | **100% 정확** |
| 일관성 | 매번 다른 표현 가능 | **항상 동일** |
| 설명 가능성 | 블랙박스 | **규칙으로 설명 가능** |
| 적합한 용도 | 창작, 요약, 대화 | **공식 보도자료** |

## 7. 시연

### 7.1 시연 시나리오

```
┌─────────────────────────────────────────────────────────────────┐
│                      시연 순서 (약 5분)                          │
└─────────────────────────────────────────────────────────────────┘

① 파일 업로드 (30초)
   - 분석표 엑셀 파일을 드래그 앤 드롭
   - "2025년 2분기" 자동 인식 확인

② 부문별 보도자료 생성 (1분)
   - "광공업생산" 클릭 → 미리보기 확인
   - 생성된 문장과 원본 데이터 비교

③ 시도별 보도자료 생성 (1분)
   - "서울" 클릭 → 9개 지표 테이블 확인
   - 막대그래프 자동 생성 확인

④ 전체 일괄 생성 (1분)
   - "전체 생성" 클릭
   - 50개 페이지 자동 생성 과정 확인

⑤ 내보내기 (1분)
   - HTML 다운로드
   - 한글(HWP)에 복사-붙여넣기 시연
```

### 7.2 핵심 포인트

시연 시 강조할 내용:

| 포인트 | 시연 방법 |
|--------|----------|
| **정확성** | 엑셀 원본 수치와 생성된 보도자료 수치 비교 |
| **자동화** | 클릭 한 번으로 전체 생성 |
| **일관성** | 동일한 파일로 여러 번 생성해도 같은 결과 |
| **수정 가능** | 생성된 내용 직접 수정 후 저장 |

## 8. 기대 효과 및 향후 계획

### 8.1 기대 효과

#### 8.1.1 실무자 현황 및 작업 프로세스

**실무팀 구성:**
- 담당 사무관 1명, 주무관 1명 (총 2명)
- 작업 기간: 약 **7일**

**현재 작업 프로세스 (AS-IS):**

```
[1-2일차] 기초자료 수집
    ↓
[3일차] 분석표 업데이트 및 집계
    ↓
[4-8일차] 한글 보도자료 초안 작성 (5일 소요) ← 병목 구간
    ↓
[9-10일차] 기자 질문 대응용 해석 작업 (2일)
```

**상세 작업 내용:**

| 단계 | 작업 내용 | 소요 시간 | 비고 |
|------|----------|----------|------|
| **1단계: 데이터 수집 및 분석표 작성** | 국가데이터처 내부 통계표 → 수집표 엑셀 업데이트 | 거의 즉시 | 엑셀 함수로 자동 계산 |
| **2단계: 분석표 집계** | 수집표 수치를 분석표 집계시트에 입력 | 거의 즉시 | 엑셀 내장 함수로 자동 계산 |
| **3단계: 보도자료 초안 작성** | 분석표 데이터 → 한글 프로그램으로 보도자료 작성 | **5일** | **수동 작업** |
| **4단계: 해석 작업** | 보도자료 발표 시 기자 질문 대응 준비 | **2일** | 분석 및 전망 |

**3단계의 문제점 (보도자료 초안 작성 5일):**

현재 한글 프로그램으로 보도자료를 작성할 때는:

1. **복사-붙여넣기 작업**
   - 분석표에서 수치를 한글 프로그램으로 일일이 복사-붙여넣기
   - 표 생성 및 서식 맞추기

2. **나레이션 작성 (수동 타이핑)**
   - 분석표를 읽고 통계 해석 문구를 사람이 직접 타이핑
   - 전국 단위에서 변동이 큰 품목을 **기여율(증가율) 순서대로 정렬**하여 나열하는 작업
   - **예시 나레이션**:
     ```
     □ 전국 광공업생산(108.2)은 반도체·전자부품(12.3%), 자동차·트레일러(7.8%), 
       의약품(5.2%) 등의 생산이 늘어 전년동분기대비 3.2% 증가
     
     ·전 국(3.2%): 반도체·전자부품(12.3%), 자동차·트레일러(7.8%), 의약품(5.2%)
     ```
   - 위와 같은 문구를 엑셀 표를 보면서 **일일이 타이핑**해야 함

3. **휴먼 에러 가능성**
   - 수치 오입력 위험
   - 복사 실수
   - 서식 불일치

#### 8.1.2 시스템 도입 후 개선 효과 (TO-BE)

**개선된 작업 프로세스:**

```
[1-2일차] 기초자료 수집
    ↓
[3일차] 분석표 업데이트 및 집계
    ↓
[3일차] 시스템 초안 생성 (약 3분) ← 자동화!
    ↓
[4일차] 한글 파일 편집 및 검토 (1일)
    ↓
[5-10일차] 해석 작업 (6일) ← 시간 확보!
```

**시간 절감 상세:**

| 단계 | AS-IS | TO-BE | 절감 효과 |
|------|-------|-------|----------|
| 보도자료 초안 작성 | **5일** | **약 3분** | **99.9% 단축** |
| 한글 편집 및 검토 | 포함됨 | **1일** | 초안 완성 후 검토만 |
| 해석 작업 | **2일** | **6일** | **3배 증가** |
| **총 작업 시간** | **7일** | **7일** | 효율 재분배 |

**핵심 개선 효과:**

1. **초안 생성 시간 혁신적 단축**
   - 기초자료 수집표 업로드 → **약 3분** 만에 초안 완성
   - 담당자는 한글 파일로 옮겨서 편집/검토만 하면 됨

2. **해석 작업 시간 3배 확보**
   - 기존: 2일 → 개선: **6일**
   - 더 깊고 넓은 해석과 통찰 가능
   - 국민들에게 **더 의미 있고 질 높은 분석과 전망** 제공

3. **업무 만족도 향상**
   - 단순 반복 작업에서 해방
   - 실수에 대한 긴장에서 벗어남
   - 업무 피로도 및 시간 비효율성 해소

#### 8.1.3 기대 효과 요약

| 효과 | 설명 |

### 8.2 배포 계획

#### 8.2.1 배포 일정

- **배포 시기**: 2025년 1월 5일 (두 번째 중간발표 이후)
- **배포 형태**: Windows 실행 가능한 독립 앱 (.exe 파일)
- **배포 방법**: PyInstaller를 사용한 단일 실행 파일로 패키징
- **배포 대상**: 국가데이터처 실무팀 (담당 사무관 1명, 주무관 1명)

#### 8.2.2 배포 후 테스트

1. **실무자 테스트 기간**
   - 배포 후 실무자들의 실제 업무 환경에서 테스트
   - 피드백 수집 및 개선 사항 반영

2. **사용자 만족도 기대**
   - 초안을 빠르게 생성하여 한글 파일로 붙여넣기만 하면 됨
   - 편집 및 검토 작업만 수행하면 되므로 **단순 반복 작업에서 해방**
   - 실수에 대한 긴장에서 벗어나 **업무 피로도 및 시간 비효율성 해소**
   - **사용자 만족도가 매우 높을 것으로 기대**

#### 8.2.3 배포 형태의 장점

**망분리 환경 대응:**
- 국가데이터처는 망분리 환경으로 일반적인 배포 방식 어려움
- PyInstaller로 단일 실행 파일 생성 → USB로 전달 가능
- 더블클릭으로 즉시 실행, 별도 설치 불필요
- 의존성 패키지 모두 포함

## 9. Q&A 예상 질문

### 질문 1: 왜 LLM을 사용하지 않았나요?

**답변:**

공식 통계 보도자료는 **0.1%의 오류도 허용되지 않습니다.**

| 구분 | LLM의 위험성 | 규칙기반의 안전성 |
|------|-------------|------------------|
| 수치 | 가끔 잘못된 숫자 생성 가능 | 원본 그대로 사용 |
| 표현 | 매번 다른 문장 생성 | 정해진 패턴 사용 |
| 검증 | 결과 예측 불가 | 규칙으로 추적 가능 |

향후 고도화 단계에서 **분석 코멘트 생성** 등 제한된 영역에 LLM 도입을 검토하고 있습니다.

---

### 질문 2: 제안서에서 요구한 KOSIS API 연동은 어떻게 됐나요?

**답변:**

프로젝트 초기에는 KOSIS API에서 직접 데이터를 수집하는 방식을 검토했으나, 업무 분석 과정에서 **더 효율적인 방법**을 발견했습니다.

| 접근 방식 | Bottom-up (KOSIS API) | Top-down (분석표) |
|----------|----------------------|-------------------|
| 데이터 소스 | 원시 데이터 | **이미 검증된 데이터** |
| 품질 보증 | 추가 검증 필요 | 담당자가 이미 검토 완료 |
| 구현 복잡도 | API 연동 + 집계 로직 필요 | 엑셀 파싱만 필요 |

국가데이터처에서는 이미 KOSIS 데이터를 집계하여 **분석표**를 만들고 있었습니다. 이 검증된 데이터를 활용하는 것이 더 안전하고 효율적입니다.

---

### 질문 3: HWP 파일로 직접 출력할 수 없나요?

**답변:**

현재는 **HTML → 한글 복사-붙여넣기** 방식을 사용합니다.

| 방식 | 설명 |
|------|------|
| **현재** | HTML 생성 → 한글에 복사-붙여넣기 |
| **향후** | python-hwp 라이브러리로 직접 HWP 생성 예정 |

복사-붙여넣기 방식도 **표와 서식이 그대로 유지**되어 실무에서 충분히 활용 가능합니다.

---

### 질문 4: 데이터가 누락되면 어떻게 되나요?

**답변:**

시스템은 **안전하게 대응**합니다:

| 상황 | 처리 방법 |
|------|----------|
| 값이 없음 | `[ ]` 빈 칸으로 표시 |
| 형식 오류 | 원본 값 그대로 출력 |
| 시트 누락 | 해당 보도자료 생성 건너뜀 |

담당자가 **육안으로 확인하고 직접 입력**할 수 있도록 빈 칸으로 표시합니다. 잘못된 값을 자동으로 만들어내는 것보다 안전합니다.

---

### 질문 5: 새로운 보도자료를 추가하려면?

**답변:**

시스템이 **모듈화되어 있어** 추가가 용이합니다:

```
새 보도자료 추가 절차:

1. 템플릿 작성 (HTML 서식)
2. 생성기 작성 (엑셀에서 데이터 추출)
3. 설정에 등록

→ 기존 시스템 수정 없이 확장 가능
```

이것이 **확장 가능한 프레임워크**로 설계한 이유입니다.

---

### 질문 6: 테스트는 어떻게 했나요?

**답변:**

| 테스트 방법 | 내용 |
|------------|------|
| **정답 비교** | 기존 보도자료(정답)와 생성 결과 비교 |
| **수치 검증** | 엑셀 원본과 생성된 수치 대조 |
| **육안 검토** | 담당자 피드백 반영 |

`correct_answer/` 폴더에 기존 보도자료 이미지를 저장하여 비교 검토했습니다.

---

### 질문 7: 배포는 어떻게 하나요?

**답변:**

국가데이터처는 **망분리 환경**으로 일반적인 방법이 어렵습니다.

| 방식 | 가능 여부 |
|------|----------|
| 클라우드 배포 | ❌ 불가 (망분리) |
| pip 설치 | ❌ 어려움 (인터넷 차단) |
| **실행 파일(.exe)** | ✅ **최적** |

PyInstaller로 **단일 실행 파일**을 만들어 USB로 전달하면:
- 더블클릭으로 즉시 실행
- 별도 설치 불필요
- 의존성 패키지 포함
