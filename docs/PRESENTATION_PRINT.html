1. 프로젝트 배경 및 목표

1.1 의뢰 기관 소개

국가데이터처는 국가 통계를 담당하는 핵심 기관으로, 66종의 승인통계를 작성하고 매월 20건 내외의 보도자료를 발표합니다.

1.2 제안 배경

국가데이터처에서 매 분기 발행하는 "지역경제동향" 보도자료는:

  • 전국 17개 시도의 경제 지표를 종합 분석
  • 10개 경제 부문(생산, 소비, 물가, 고용 등)을 다룸
  • 약 78페이지 분량으로 구성 (50개 보도자료 항목)
  • 작성에 전체 인력이 약 1주일 투입

1.3 제안서 요구사항

"AI를 이용한 통계자료 기반 보도자료 자동작성 시스템 개발"
— 11기 캡스톤 프로젝트 제안서 (국가데이터처)
요구사항설명
데이터 전처리엑셀 통계 자료 자동 로딩
통계표 자동 생성데이터 기반 표 작성

템플릿 모듈화재사용 가능한 구조

3. 해결 방안

3.1 접근 방식 비교

프로젝트 초기에 두 가지 접근 방식을 검토했습니다.

구분LLM (대규모 언어모델)규칙기반 시스템
장점자연스러운 문장, 유연한 대응100% 정확도, 일관된 출력
단점수치 오류 가능(hallucination), 결과 예측 불가새로운 패턴 대응에 개발 필요
적합성창의적 글쓰기, 요약통계 보도자료 (정확성 필수)

3.2 왜 규칙기반을 선택했는가?

공식 보도자료의 특성:

  1. 정확성이 생명 — 통계 수치가 하나라도 틀리면 정정 보도 필요
  2. 정형화된 패턴 — 매 분기 동일한 구조와 표현 사용
  3. 검증 가능성 — 어떤 규칙으로 생성되었는지 추적 가능해야 함
"0.1%의 오류도 허용되지 않는 공식 보도자료에서는 LLM의 hallucination 위험을 감수할 수 없습니다."

5. 핵심 기술: 규칙기반 자연어 생성

5.1 규칙기반 자연어 생성이란?

사전에 정의된 규칙에 따라 데이터를 문장으로 변환하는 기술입니다.

┌─────────────────────────────────────────────────────────────────┐ │ 규칙기반 자연어 생성 원리 │ └─────────────────────────────────────────────────────────────────┘ [원본 데이터] [규칙 적용] [결과 문장] │ │ │ 증감률: 3.5% ───▶ 증가/감소 판단 ───▶ "3.5% 증가" 지역: 경기 ───▶ 지역명 매핑 ───▶ "경기" 순위: 1위 ───▶ 순위 표현 선택 ───▶ "가장 높았다"

5.2 적용된 6가지 규칙 유형

규칙 1: 매핑 규칙 (이름 변환)

엑셀의 긴 공식 명칭을 보도자료용 간결한 표현으로 변환합니다.

엑셀 원본 (31자)보도자료 표기 (7자)
전자 부품, 컴퓨터, 영상, 음향 및 통신장비 제조업반도체·전자부품
의료, 정밀, 광학 기기 및 시계 제조업의료·정밀
자동차 및 트레일러 제조업자동차

※ 총 28개 업종, 17개 지역에 대한 매핑 규칙 정의

규칙 2: 계산 규칙 (수치 처리)

규칙설명예시
소수점 처리첫째 자리까지 반올림3.456 → 3.5
백분율 변환소수를 퍼센트로0.035 → 3.5%
천 단위 구분쉼표 추가1234567 → 1,234,567

규칙 3: 정렬/분류 규칙

분류 기준설명
증감률 순높은 순서대로 나열
지역 그룹수도권/비수도권 구분
상위 N개1~3위 업종만 추출
7

규칙 4: 조건부 텍스트 생성

수치의 부호나 크기에 따라 다른 표현을 사용합니다.

조건생성되는 텍스트
증감률 > 0"X.X% 증가하였다"
증감률 < 0"X.X% 감소하였다"
증감률 = 0"전분기와 동일하였다"
1위 업종"가장 높은 증가율을 보였다"
전국 평균 이상"전국 평균을 상회하였다"
전국 평균 이하"전국 평균을 하회하였다"

규칙 5: 문장 템플릿

정해진 문장 구조에 데이터를 삽입합니다.

템플릿: "{지역}의 {지표}은(는) 전분기 대비 {증감률}% {증감표현}하였다."

결과: "경기의 광공업생산은 전분기 대비 3.5% 증가하였다."

규칙 6: 결측치 처리

데이터가 없는 경우 안전하게 처리합니다.

상황처리 방법
값이 없음"[ ]" 플레이스홀더 표시
수식 오류기본값 또는 경고 메시지
형식 오류원본 값 그대로 표시

※ 담당자가 직접 확인하고 수정할 수 있도록 빈 칸으로 표시

5.3 규칙기반의 장점

특성규칙기반의 장점
정확성정의된 규칙대로만 동작 → 오류 없음
일관성동일한 입력 → 항상 동일한 출력
추적성왜 이런 문장이 나왔는지 규칙으로 설명 가능
수정 용이규칙만 수정하면 전체 출력에 반영

8.1 기대 효과

8.1.1 실무자 현황 및 작업 프로세스

실무팀 구성: 담당 사무관 1명, 주무관 1명 (총 2명) | 작업 기간: 약 7일

현재 작업 프로세스 (AS-IS):

[1-2일차] 기초자료 수집
↓
[3일차] 분석표 업데이트 및 집계
↓
[4-8일차] 한글 보도자료 초안 작성 (5일 소요) ← 병목 구간
↓
[9-10일차] 기자 질문 대응용 해석 작업 (2일)
단계작업 내용소요 시간비고
1단계데이터 수집 및 분석표 작성거의 즉시엑셀 함수로 자동 계산
2단계분석표 집계거의 즉시엑셀 내장 함수로 자동 계산
3단계보도자료 초안 작성5일수동 작업
4단계해석 작업2일분석 및 전망

3단계의 문제점:

  • 복사-붙여넣기 작업: 분석표에서 수치를 한글 프로그램으로 일일이 복사-붙여넣기, 표 생성 및 서식 맞추기
  • 나레이션 작성 (수동 타이핑): 분석표를 읽고 통계 해석 문구를 사람이 직접 타이핑
    예시:
    □ 전국 광공업생산(108.2)은 반도체·전자부품(12.3%), 자동차·트레일러(7.8%),
      의약품(5.2%) 등의 생산이 늘어 전년동분기대비 3.2% 증가

    ·전 국(3.2%): 반도체·전자부품(12.3%), 자동차·트레일러(7.8%), 의약품(5.2%)

    전국 단위에서 변동이 큰 품목을 기여율(증가율) 순서대로 정렬하여 나열하는 작업을 일일이 타이핑

8.2 배포 계획

8.2.1 배포 일정

  • 배포 시기: 2025년 1월 5일 (두 번째 중간발표 이후)
  • 배포 형태: Windows 실행 가능한 독립 앱 (.exe 파일)
  • 배포 방법: PyInstaller를 사용한 단일 실행 파일로 패키징
  • 배포 대상: 국가데이터처 실무팀 (담당 사무관 1명, 주무관 1명)

8.2.2 배포 후 테스트 및 사용자 만족도

  • 실무자들의 실제 업무 환경에서 테스트
  • 초안을 빠르게 생성하여 한글 파일로 붙여넣기만 하면 됨
  • 편집 및 검토 작업만 수행 → 단순 반복 작업에서 해방
  • 실수에 대한 긴장에서 벗어나 업무 피로도 및 시간 비효율성 해소
  • 사용자 만족도가 매우 높을 것으로 기대

8.2.3 배포 형태의 장점

망분리 환경 대응: 국가데이터처는 망분리 환경으로 일반적인 배포 방식 어려움

  • PyInstaller로 단일 실행 파일 생성 → USB로 전달 가능
  • 더블클릭으로 즉시 실행, 별도 설치 불필요
  • 의존성 패키지 모두 포함

8.3 향후 발전 방향

단기 (1~3개월)

  • HWP 직접 출력 기능
  • 데이터 검증 자동화 강화

중기 (3~6개월)

  • 차트 이미지 자동 생성
  • 승인 워크플로우 구현

장기 (6개월~1년)

  • 다른 보도자료로 확장
    (인구동향, 물가동향 등)
  • AI 기반 분석 코멘트 생성
    (LLM 도입 검토)

8.4 확장 가능성

본 시스템의 구조(규칙기반 + 템플릿)는 다른 보도자료에도 적용 가능합니다:

확장 대상적용 방법
인구동향 보도자료새 템플릿 + 생성기 추가
물가동향 보도자료새 템플릿 + 생성기 추가
고용동향 보도자료새 템플릿 + 생성기 추가

9. Q&A 예상 질문

Q1. 왜 LLM을 사용하지 않았나요?

공식 통계 보도자료는 0.1%의 오류도 허용되지 않습니다.

구분LLM의 위험성규칙기반의 안전성
수치가끔 잘못된 숫자 생성 가능원본 그대로 사용
표현매번 다른 문장 생성정해진 패턴 사용
검증결과 예측 불가규칙으로 추적 가능

향후 고도화 단계에서 분석 코멘트 생성 등 제한된 영역에 LLM 도입을 검토하고 있습니다.

Q2. KOSIS API 연동은 어떻게 됐나요?

업무 분석 과정에서 더 효율적인 방법을 발견했습니다.

접근 방식Bottom-up (KOSIS API)Top-down (분석표)
데이터 소스원시 데이터이미 검증된 데이터
품질 보증추가 검증 필요담당자가 이미 검토 완료

국가데이터처에서는 이미 KOSIS 데이터를 집계하여 분석표를 만들고 있었습니다.

Q3. HWP 파일로 직접 출력할 수 없나요?

현재는 HTML → 한글 복사-붙여넣기 방식을 사용합니다.

복사-붙여넣기 방식도 표와 서식이 그대로 유지되어 실무에서 충분히 활용 가능합니다.

향후 python-hwp 라이브러리로 직접 HWP 생성 예정입니다.

Q4. 데이터가 누락되면 어떻게 되나요?

시스템은 안전하게 대응합니다:

  • 값이 없음 → [ ] 빈 칸으로 표시
  • 형식 오류 → 원본 값 그대로 출력
  • 시트 누락 → 해당 보도자료 생성 건너뜀

담당자가 육안으로 확인하고 직접 입력할 수 있도록 빈 칸으로 표시합니다.

Q5. 새로운 보도자료를 추가하려면?

시스템이 모듈화되어 있어 추가가 용이합니다:

  1. 템플릿 작성 (HTML 서식)
  2. 생성기 작성 (엑셀에서 데이터 추출)
  3. 설정에 등록

→ 기존 시스템 수정 없이 확장 가능 (확장 가능한 프레임워크)

테스트 방법내용
정답 비교기존 보도자료(정답)와 생성 결과 비교

Q7. 배포는 어떻게 하나요?

국가데이터처는 망분리 환경으로 일반적인 방법이 어렵습니다.

방식가능 여부
클라우드 배포❌ 불가 (망분리)
pip 설치❌ 어려움 (인터넷 차단)
실행 파일(.exe)✅ 최적

PyInstaller로 단일 실행 파일을 만들어 USB로 전달하면:

  • 더블클릭으로 즉시 실행
  • 별도 설치 불필요
  • 의존성 패키지 포함
