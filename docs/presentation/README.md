# 캡스톤 프로젝트 발표 연습 자료

이 폴더에는 20분 PT 발표를 위한 일주일간 연습 자료가 포함되어 있습니다.

## 파일 구조

```
presentation/
├── README.md                       # 이 파일 (전체 가이드)
├── presentation_structure.md       # 발표 구조 및 시간 배분
├── rule_based_implementation.md    # ⭐ 규칙기반 구현 상세 (가장 중요!)
├── module_explanation.md           # 모듈별 역할 및 구현 방법
├── key_cards_template.md           # 슬라이드별 핵심 카드 템플릿
├── expected_qa.md                  # 예상 Q&A 리스트
├── tough_questions.md              # 까다로운 질문 대비
├── challenges.md                   # 개발 과정의 도전과 해결
├── config_explanation.md           # 설정 파일 설명
├── day_by_day_guide.md            # 일별 발표 연습 가이드
└── practice_checklist.md           # 일주일간 연습 체크리스트
```

### ⭐ 핵심 문서: 규칙기반 구현

**`rule_based_implementation.md`**는 발표의 가장 중요한 내용입니다.

포함 내용:
- 왜 규칙기반을 선택했는가? (LLM vs 규칙기반)
- 3계층 구조 (Generator → Schema → Template)
- 매핑 규칙, 계산 규칙, 정렬 규칙, 텍스트 생성 규칙
- 코드 예시 및 발표 시나리오

## 사용 방법

### 1단계: Day 1 - 스크립트 전환 및 구조 파악
1. `presentation_structure.md` 읽기
2. `key_cards_template.md`를 기반으로 본인의 핵심 카드 작성
3. 전체 구조 머릿속에 정리

### 2단계: Day 2-7 - 단계별 연습
1. `practice_checklist.md`의 해당 날짜 체크리스트 확인
2. 각 날짜별 연습 내용 수행
3. 체크리스트 완료 확인

### 3단계: Q&A 대비
1. `expected_qa.md` 읽기
2. 각 질문에 대한 답변 연습
3. 예상하지 못한 질문 대비

## 핵심 원칙

1. **스크립트는 참고용**
   - 문장 암기 X
   - 핵심 아이디어 이해 O

2. **자연스러운 대화**
   - 친구에게 설명하는 느낌
   - 청중과 소통하는 느낌

3. **슬라이드는 도구**
   - 벽 보지 않기
   - 청중과 눈맞춤

4. **완벽보다 자연스러움**
   - 작은 실수는 OK
   - 자연스러운 멈춤도 OK

## 성공 기준

- [ ] 전체 발표 18-20분 내 완료
- [ ] 슬라이드 의존도 최소화 (참고만)
- [ ] 청중과 자연스러운 눈맞춤
- [ ] 스크립트 없이도 핵심 전달 가능
- [ ] 자신감 있는 발표

## 추가 팁

### 눈맞춤 연습
- 거울 앞에서 연습
- 슬라이드 2초 → 청중 10초
- Z자 스캔 패턴

### 자연스러운 말하기
- 핵심 카드만 보기
- 매번 다른 표현 시도
- 연결어 활용 ("그래서", "이제", "다음으로")

### 긴장 관리
- 4-7-8 호흡법
- 파워 포즈
- 긍정 시각화
