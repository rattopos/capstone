## 1. 도입부 (3분)

### 목표
- 첫인상이 좋은 도입
- 문제의 중요성 강조
- 프로젝트 가치 어필

### 핵심 메시지
1. **문제 정의** (30초)
   - 국가데이터처의 분기별 보도자료 작성 업무
   - 수작업으로 수일 소요
   - 오타 및 숫자 오류 발생

2. **프로젝트 소개** (1분)
   - 지역경제동향 보도자료 자동 생성 시스템
   - 의뢰 기관: 국가데이터처
   - 개발 기간: 2주 (12/11 ~ 12/26)

3. **프로젝트 목표** (1분 30초)
   - 작업 시간 단축: 수일 → 분 단위
   - 오류 감소: 수기 입력 오류 방지
   - 표준화: 일관된 포맷
   - 재현성: 동일 입력 → 동일 출력

### 전환어
"그렇다면, 이 시스템이 어떻게 작동하는지 살펴보겠습니다."

---

## 2. 본론 1: 시스템 아키텍처 및 주요 기능 (7분)

### 목표
- 시스템 구조를 쉽게 설명
- 50개 보도자료 생성 능력 강조
- 핵심 기능 명확히 전달

### 핵심 메시지

#### 2-1. 시스템 아키텍처 (2분)
- Flask 웹 애플리케이션
- 모듈화된 구조 (routes, services, utils, config)
- 입력: 분석표 엑셀, 기초자료 수집표
- 출력: HTML, HWPX, Excel, PDF

#### 2-2. 주요 기능 - 50개 보도자료 (3분)
- **요약 보도자료 9개**: 표지, 일러두기, 목차, 인포그래픽, 요약 5개
- **부문별 보도자료 10개**: 광공업생산, 서비스업생산, 소비동향, 건설동향, 수출입, 물가, 고용/실업, 인구이동
- **시도별 보도자료 18개**: 17개 시도 + GRDP 참고
- **통계표 13개**: 각 부문별 통계표 + 부록

#### 2-3. 웹 대시보드 기능 (2분)
- 드래그 앤 드롭 파일 업로드
- 실시간 미리보기
- 4개 탭 구조 (요약/부문별/시도별/통계표)
- 결측치 시각화 (노란색 형광펜)
- 다중 포맷 내보내기

### 전환어
"이제 이 시스템의 기술적 성과를 살펴보겠습니다."

## 3. 본론 2: 기술적 성과 및 데모 (7분)

### 목표
- 기술적 성과를 임팩트 있게 전달
- 데모 시연 자연스럽게 진행
- 숫자와 통계 효과적으로 활용

### 핵심 메시지

#### 3-1. 기술적 성과 (4분)

**1. 스키마 기반 데이터 추출**
- JSON 스키마로 데이터 매핑 정의
- 39개 스키마 파일
- 새로운 보도자료 추가 시 스키마만 작성

**2. 가중치 기반 순위 시스템**
- 17개 시도 경제지표 실시간 분석
- 증감률에 가중치 적용
- 상위/하위 지역 자동 하이라이트

**3. 결측치 처리 시스템**
- 노란색 형광펜으로 시각화
- 사용자 직접 수정 가능
- 모달창으로 한눈에 확인

**4. 다중 출력 포맷 지원**
- HTML (미리보기)
- HWPX (한글 문서)
- Excel (분석표 자동 생성)
- PDF (인쇄용)

**5. 모듈화된 코드 구조**
- 104개 커밋
- 71개 HTML 템플릿
- 15개 Python 생성기
- 39개 JSON 스키마

#### 3-2. 데모 시연 (3분)
- 대시보드 화면 보여주기
- 파일 업로드 → 미리보기 → 내보내기 흐름
- "지금 보시는 것은..." 자연스러운 설명

### 전환어
"마지막으로 향후 개선 과제를 말씀드리겠습니다."

- 강렬한 마무리
- 프로젝트 가치 재강조

### 핵심 메시지

#### 4-1. 프로젝트 성과 요약 (1분)

- 작업 시간: 수일 → 분 단위
