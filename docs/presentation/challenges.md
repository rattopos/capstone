# 프로젝트 수행 중 어려운 점 및 극복 과정

## 1. 비공개 자료로 인한 데이터 접근 제한

### 📌 상황
- 국가데이터처의 **분석표, 기초자료 수집표** 등 핵심 데이터가 비공개 자료
- 외부에서 실제 데이터에 접근하기 어려운 환경
- 개발 및 테스트에 필요한 실제 데이터 확보의 어려움

### 😰 어려웠던 점
- 실제 분석표 구조를 파악하기 위한 자료 확보 지연
- 데이터 없이 템플릿/파싱 로직을 먼저 개발해야 하는 상황
- 정확도 검증을 위한 실제 데이터 부족
- 보안상 외부 환경에서 테스트 제약

### 💪 극복 방법
- 국가데이터처 담당자와 **주기적인 미팅**을 통해 자료 확보
- 제공받은 **샘플 데이터**를 기반으로 개발 진행
- 비공개 데이터 구조를 분석하여 **스키마 문서화**
- 정답 이미지(correct_answer/)를 참고하여 출력 형식 파악

### ✅ 결과
- 제한된 자료 환경에서도 시스템 개발 완료
- 실제 분석표와 호환되는 파싱 로직 구현
- 실무 환경의 **보안/비공개 정책** 대응 경험 확보

---

## 2. 팀원 중도포기로 인한 1인 프로젝트 전환

### 📌 상황
- 프로젝트 초기 **3인 팀**으로 구성하여 시작
- 1차 중간발표 직전/직후 시점에 **팀원 2명이 중도포기**
- 이후 **단독(1인) 프로젝트**로 전환하여 진행

### 😰 어려웠던 점
- 원래 3인이 분담할 예정이던 작업량을 혼자 수행해야 함
- 프론트엔드, 백엔드, 데이터 처리 등 전 영역을 단독으로 담당
- 역할 분담 계획이 무산되어 프로젝트 일정 재조정 필요
- 기술적 의사결정 시 논의할 팀원 부재

### 💪 극복 방법
- 핵심 기능 우선순위를 재정립하고 집중 개발
- Flask 기반 웹 애플리케이션으로 프론트엔드/백엔드 통합 개발
- 템플릿 엔진(Jinja2) 활용으로 개발 효율성 확보
- 일정 관리를 철저히 하여 마감 기한 준수

### ✅ 결과
- **1인 풀스택 개발**로 프로젝트 완수
- 엑셀 파싱, 데이터 처리, 보고서 생성, 웹 UI 전체 구현
- 위기 대응 능력 및 자기주도적 문제해결 역량 향상

## 3. 지역경제동향 보고서 서식 변경

### 📌 상황
- 프로젝트 타겟: 국가데이터처 '지역경제동향' 보고서 자동화
- 프로젝트 진행 중 **2024년 4분기부터 보고서 서식이 변경**됨
- 기존에 개발한 템플릿과 로직을 신규 서식에 맞게 수정 필요

### 😰 어려웠던 점
- 이미 구현된 HTML 템플릿 구조 재설계 필요
- 데이터 매핑 로직 수정 작업 발생
- 서식 변경 시점을 사전에 예측할 수 없었음
- 변경된 서식에 대한 공식 가이드라인 부재

### 💪 극복 방법
- **템플릿 기반 설계**로 서식 변경에 유연하게 대응
  - Jinja2 템플릿을 모듈화하여 수정 범위 최소화
  - 데이터와 표현 계층 분리로 유지보수성 확보
- 변경된 실제 보고서를 분석하여 신규 서식 파악
- 점진적 수정으로 기존 기능 유지하면서 업데이트

### ✅ 결과
- 서식 변경에 성공적으로 대응하여 최신 양식 지원
- **확장 가능한 템플릿 구조** 구축으로 향후 변경에도 대응 가능
- 실무 환경과 동일한 불확실성 대응 경험 확보

## 📝 보고서 기술 예시

### 프로젝트 수행 중 어려운 점

1. **비공개 자료 접근 제한**
   - 국가데이터처의 분석표, 기초자료 수집표 등 핵심 데이터가 비공개 자료로 분류되어 있어 외부에서 실제 데이터에 접근하기 어려웠음
   - 담당자와의 주기적인 미팅을 통해 샘플 데이터를 확보하고, 정답 이미지를 참고하여 개발을 진행함

2. **팀 구성 변경**
   - 프로젝트 초기 3인 팀으로 구성하였으나, 1차 중간발표 시점에 팀원 2명이 중도포기하여 이후 1인 프로젝트로 전환하여 진행하였음
   - 원래 역할 분담하여 진행할 예정이던 프론트엔드, 백엔드, 데이터 처리 등 전 영역을 단독으로 수행함

3. **외부 환경 변화**
   - 프로젝트 진행 중 국가데이터처의 '지역경제동향' 보고서 서식이 2024년 4분기부터 변경됨
   - 이에 따라 기존 개발 내용을 신규 서식에 맞게 재설계 및 수정하는 작업이 추가로 발생함
